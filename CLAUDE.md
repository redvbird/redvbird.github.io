# CLAUDE.md — 嗣通信息官网项目

## 📌 项目概述

- **项目名称**:上海嗣通信息技术有限公司官方网站
- **域名**:sitongservice.com
- **类型**:B2B 多页营销站(共 21 个页面,2026-05-13 由单页升级为多页结构以利百度 SEO)
- **核心目标**:获客转化——让上海地区的中小企业了解 IT 外包服务并发起咨询

**目标用户画像**:
- 上海本地中小企业主(20-200 人规模)
- 企业 IT 采购负责人、行政总监
- 没有专职 IT 团队、需要外包服务的公司
- 主要通过百度搜索找到网站

**核心服务**(与 index.html `#services` 同步):
- 企业 IT 运维外包(桌面运维、邮箱与域账号、补丁与杀毒)
- 服务器与网络设备托管(机房巡检、IDC 托管、防火墙/交换机配置)
- 办公设备采购与维护(电脑、打印机、显示器、网络设备)
- 弱电工程(网络布线、监控安防、门禁、视频会议系统)

---

## 🛠 技术栈

- **HTML5** — **多页结构**,每个页面独立 HTML(共 20 页),共享 header/footer/nav 通过复制粘贴维护
- **CSS3** — 纯手写,无预处理器,统一 `/css/style.css`
- **JavaScript** — 原生 JS,无框架依赖,统一 `/js/main.js`
- **字体**:思源黑体(本地或国内 CDN 自托管)

**明确不使用**:
- ❌ React / Vue / Angular 等前端框架
- ❌ 海外 CDN(Google Fonts、jsDelivr 海外节点等)
- ❌ Node.js / PHP 构建流程
- ❌ Bootstrap、Element UI 等厚重 UI 库

**理由**:小站,必须保证国内访问速度与 SEO 干净。Header/footer 改动由 AI 助手用 `Edit replace_all` 批量更新,人工不直接维护。

## 💻 代码规范

### HTML
- 使用语义化标签:`<header> <nav> <main> <section> <article> <footer>`
- 避免 `<div>` 套娃
- 所有 `<img>` 必须有 `alt` 和明确的 `width / height`(防止 CLS)

### CSS
- 类名使用 BEM:`.service-card__title--featured`
- 颜色、字号、间距用 CSS 变量在 `:root` 统一管理
- **不写 inline style**
- 媒体查询断点:`768px`(平板)、`1024px`(桌面)

### JavaScript
- 原生 JS,不引入 jQuery
- 函数与变量用 camelCase
- 避免污染全局命名空间(IIFE 或模块化)
- 营销页 JS 主要做交互动效,不写复杂业务逻辑
